Overview:

This document provides a quick reference to the CGI specifications for interfacing with the bigbangonline.org server. These are meant to help in understanding what data is expected and possibly why an error occurred.

Property Descriptions
Multiple properties are normally a comma-separated list. These descriptions are not
necessarily of data the user enters but might be data from the server or the Java package.
Name Description Match Pattern Sample Value

ADDRESS Address of user's institution. May contain ASCII(8) which should be interpreted as newlines [\w -\.\+\10]+ 1 Duke's way

BBN_SIM_COMMAND Available command is "run bbn simulator". The syntax for "run bbn simulator" is "run bbn simulator [Monte Carlo with $VALUE trials and uncertainties "$PATH"] [with $VARIABLE as $VALUE]... [and vary $VARIABLE [as $VALUE]... ]..." where $VARIABLE is a property obtained from GET BBN SIM TYPES. A example of this command is "run bbn simulator Monte Carlo with 10 trials and uncertainties "/SHARED/default.rate_uncer" with INITIAL_TIMESTEP as 1 with FINAL_TEMPERATURE as 1.0E+09 and vary ETA as 0.1E-10 as 1.0E-9 and vary COSMOLOGICAL_CONSTANT as 1.0 as 2.0" run bbn simulator( Monte Carlo with \d+ trials and uncertainties "[\w \.\+-/]+")? ( with \w+ as [-\+]?\d*\.?\d+([Ee][-\+]?\d+)?)* ( and vary \w+( as [-\+]?\d*\.?\d+([Ee][-\+]?\d+)?)+)* run bbn simulator with INITIAL_TIMESTEP as 1 with FINAL_TEMPERATURE as 1.0E-09 and vary ETA as 0.1E-10 as 1.0E-9 and vary COSMOLOGICAL_CONSTANT as 1.0 as 2.0
BIBLIO_STRING Reaclib biblio code [\w\+-]{1,4} cf88

Administration
Multiple properties are normally a comma-separated list. These descriptions are
of options used for the administrative functions of BBN.
Request Description Submit Properties Response Properties
GET ID
 Get session ID from server HEADER,ACTION,USER,PW ID
GET TIMEOUT
 Users are automatically logged out if a CGI request is not called within a certain timeframe. This request returns the length of the timeframe in minutes. HEADER,ID,ACTION,USER,PW TIMEOUT
LOGOUT
 Terminates user's session HEADER,ID,ACTION,USER,PW LOGOUT
REGISTER
 Submit a user registration for approval for an account. The NOTES field allows additional information (supervisor / research mentor) to be submitted HEADER, ID, ACTION, USER, PW, LAST_NAME , FIRST_NAME, EMAIL, INSTITUTION, ADDRESS, COUNTRY, RESEARCH_TYPE, DESIRED_USERNAME, DESIRED_PASSWORD, PASSWORD_HINT, NOTES, HEAR_OF_SUITE REGISTER

BBN Simulator
Request Description Submit Properties Response Properties
GET BBN SIM TYPES
 Get list of available BBN simulations and all default values. One set of properties is returned for every simulation type. Each line contains a property listed in the response property list. The first property in the set will always be the SIMULATION_TYPE. HEADER, ID, ACTION, USER, PW SIMULATION_TYPE, DESCRIPTION, TIME_STEP_CONSTANT1, TIME_STEP_CONSTANT2, INITIAL_TIMESTEP, INITIAL_TEMPERATURE, FINAL_TEMPERATURE, SMALLEST_ABUND_ALLOWED, ACCUMULATION_INCREMENT, GRAVITATIONAL_CONSTANT, NEUTRON_LIFETIME, NUMBER_NEUTRINO_SPECIES, ETA, COSMOLOGICAL_CONSTANT, XI_ELECTRON, XI_MUON, XI_TAUON
GET BBN LOOPING DEFAULTS
 Get looping defaults for bbn simulator. One set of properties is returned for every quantity in PARAMETERS. Each line contains a property listed in the response property list. The first property in the set will always be the PARAMETER. HEADER, ID, ACTION, USER, PW, PARAMETERS PARAMETER, MIN, MAX, INCREMENT_LIN, INCREMENT_LOG, LOWER_BOUND, UPPER_BOUND
BBN SIM SETUP
Uses default library 		Prepare library for a BBN simulation HEADER, ID, ACTION, USER, PW, SIMULATION_TYPE, LIBRARY REPORT, SUMMARY, MIN_ISOTOPE, MAX_ISOTOPE
RUN BBN SIM
 HEADER, ID, ACTION, USER, PW, BBN_SIM_COMMAND RUN
BBN SIM UPDATE
 Returns the latest output from bbn simulator. It only sends output that has not been sent before and returns as many lines of text that will fit within 1000 characters. Newlines will be replaced with ASCII(8) so that the TEXT property will fit on one line HEADER, ID, ACTION, USER, PW TEXT_SKIPPED, SIMULATION, TEXT, TOTAL_RUNS, CURRENT_RUN
ABORT BBN SIM
 HEADER, ID, ACTION, USER, PW STOP
SAVE BBN SIM
 HEADER, ID, ACTION, USER, PW, PATH, OVERWRITE, NOTES REPORT

BBN Visualizer
Request Description Submit Properties Response Properties
GET BBN RUN LIST
 Get list of folders and simulations in list of PATHS. One set of properties is returned for every PATH in PATHS. Each line contains a property listed in the response property list. The first property in the set will always be PATH. HEADER, ID, ACTION, USER, PW, PATHS PATH, RUNS, FOLDERS
GET BBN RUN INFO
 Get list of properties set in the BBN_SIM_COMMAND for a list of PATHS. One set of properties is returned for every PATH in PATHS. Each line contains a property listed in the response property list. The first property in the set will always be PATH. HEADER, ID, ACTION, USER, PW, PATHS PATH, NOTES, CREATION_DATE, MODIFICATION_DATE, MONTE_CARLO_LIST, LOOPING_LIST, LIBRARY, RATE_UNCERTAINTY_PATH, MONTE_CARLO_TRIALS, RECIPE
GET BBN RUN DATA
 Get data from a bbn simulation. One set of properties is returned for every table of data meeting the conditions in GET_BBN_DATA_COMMAND. Each item in PARAMETERS corresponds with a column in DATA. The first property returned for each table will always be PATH. HEADER, ID, ACTION, USER, PW, GET_BBN_DATA_COMMAND PATH, PARAMETERS, CONSTRAINTS, DATA
BBN RUN EXIST
 Check if path to a bbn simulation is used. One set of properties is returned for every path in PATHS. HEADER, ID, ACTION, USER, PW, PATHS PATH, EXISTS
ERASE BBN RUN
 Erase bbn run for a given path HEADER, ID, ACTION, USER, PW, PATH REPORT
COPY BBN RUN TO SHARED
 Copy bbn run for a given path to SHARED folder HEADER, ID, ACTION, USER, PW, PATH REPORT
